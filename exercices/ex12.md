# Exercice 12 : tests unitaires avec *Mocha* et *node:assert*

## Énoncé

1. Avec l'approche [TDD](https://www.ionos.fr/digitalguide/sites-internet/developpement-web/quest-ce-que-le-test-driven-development/) (pour simplifier, écrivez vos tests en premier avant d'écrire vos fonctions métiers), écrivez des tests avec Mocha pour tester les fonctions mathématiques suivantes à implémenter :
- La somme entre 2 nombres
- La division entre 2 nombres
  - Si le diviseur est égal à 0, votre fonction déclenche une exception à l'aide de l'instruction suivante
  
    ```js
    throw new Error('division by 0 impossible')
    ```
  
    Donc dans votre test, vous devez vous attendre à récupérer un objet de type Error avec le message d'erreur "division by 0 impossible"
  
  - Si le diviseur est différent de 0, l'opération s'effectue
- Refactoriser la somme pour faire cette opération avec plusieurs nombres (minimum 2 nombres, pas de maximum)
